# Steps to reproduce:

- Set a breakpoint in the test file
- Run the tests with webstorm debugger (e.g. via the `test` script in package.json)
- The debugger will stop at the breakpoint
- Continue the execution
- Press `Re-Run` button in the debugger

Sometimes you will have to press `Re-Run` multiple times to see the error
